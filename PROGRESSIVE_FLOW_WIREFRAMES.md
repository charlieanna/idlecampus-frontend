### Responsive Breakpoints:
- **Mobile**: 320px - 767px (single column, stacked)
- **Tablet**: 768px - 1023px (2 columns, condensed)
- **Desktop**: 1024px+ (3 columns, full layout)

### Mobile-Specific Features:
1. **Bottom Navigation**: Persistent nav bar at bottom
2. **Expandable Sections**: Accordion-style collapse/expand
3. **Swipe Gestures**: Swipe between levels, tracks
4. **Pull to Refresh**: Update leaderboard, progress
5. **Touch Targets**: Minimum 48px for all interactive elements
6. **Simplified Tables**: Card-based layout instead of tables

## Implementation Notes

### Data Fetching Strategy:
1. **Dashboard**: Fetch user stats, track progress, recommended challenges (1 API call)
2. **Challenge List**: Paginated, 20 per page with infinite scroll
3. **Leaderboard**: Real-time updates every 2 minutes via WebSocket
4. **Progress Tracking**: Cache locally, sync on change

### Performance Optimizations:
1. Virtual scrolling for long lists (challenges, leaderboard)
2. Lazy load images and non-critical sections
3. Debounce search/filter inputs (300ms)
4. Prefetch next likely page on hover
5. Memoize expensive calculations (XP, rankings)

### Accessibility:
1. ARIA labels for all interactive elements
2. Keyboard navigation (Tab, Enter, Esc)
3. Screen reader announcements for progress updates
4. Color contrast ratio 4.5:1 minimum
5. Focus indicators visible at all times

### Animation Guidelines:
1. Level progression: Smooth line animation (500ms ease-in-out)
2. XP gain: Counter animation with particle effects
3. Achievement unlock: Modal with celebration animation
4. Progress bars: Animated fill (300ms ease-out)
5. Page transitions: Fade (200ms)
